# zigoricko • bio

Личная bio-страница в виде одностраничного сайта с плеером, эффектом дождя/грозы и панелью настроек. Стилизовано под тему `0xdead.code` / InfoSec.

## Стек

- Чистый **HTML + CSS + JS** (без сборщиков и фреймворков), всё в одном `index.html`
- **PWA**: `manifest.json` + `sw.js` (service worker) — сайт можно установить как приложение и часть контента доступна офлайн
- Шрифты: JetBrains Mono, Inter (Google Fonts)
- Иконки: Font Awesome (через CDN)

## Структура файлов

Все файлы лежат в корне репозитория (без подпапок — так удобнее заливать через веб-интерфейс GitHub по одному файлу):

```
zigoricko-bio/
├── index.html                    # вся страница: разметка, стили и логика
├── manifest.json                  # PWA-манифест (иконка, тема, standalone-режим)
├── sw.js                           # service worker: кеширование оболочки и обложек
├── avatar.jpg                      # аватар / иконка
├── thunder.mp3                      # звук грома для фоновой анимации
├── track1.mp3 … track24.mp3         # треки плейлиста
└── track1.jpg … track24.jpg          # обложки треков (имя совпадает с номером трека)
```

## Что внутри `index.html`

### Разметка и стили
- Экран загрузки (`#loader`)
- Фоновая анимация дождя и молний (`#rain`, `#lightning`, `#stormClouds`, `#boltLayer`), со звуком грома
- Блок "обо мне" (`#aboutText`)
- Аудиоплеер: обложка трека, прогресс-бар, кнопки play/prev/next, режим повтора
- Панель плейлиста с поиском по трекам (`#playlistPanel`, `#playlistSearch`)
- Панель настроек (`#settingsPanel`)

### Логика (JS)
- Плейлист из 24 треков — массив объектов `{ src, title }`
- Обложка подтягивается автоматически: у `trackN.mp3` берётся обложка `trackN.jpg` из того же каталога
- Управление воспроизведением, прогресс-бар, форматирование времени
- Кроссфейд между треками при переключении (`CROSSFADE_MS`, второй скрытый `<audio>`-элемент для плавного перехода)
- Режимы повтора (off / one / all)
- Поиск и фильтрация плейлиста
- Визуализация звука (`#visualizer`)

### `sw.js` — офлайн-режим
- Кеширует "оболочку" приложения (HTML, аватар, манифест, все обложки треков) при установке
- Аудиофайлы (`.mp3`) всегда пытается получить из сети напрямую, кеш — только как запасной вариант
- При офлайн-навигации отдаёт закешированный `index.html`

## Как запустить

Файл статический, специальный сервер не обязателен — но для корректной работы service worker и PWA-манифеста браузер должен раздавать файлы по `http(s)`, а не через `file://`. Проще всего:

```bash
npx serve .
# или
python3 -m http.server
```

Затем открыть `http://localhost:<порт>/index.html`.

Также страница будет полностью рабочей и на GitHub Pages — достаточно включить Pages в настройках репозитория (Settings → Pages → Deploy from branch → main).
